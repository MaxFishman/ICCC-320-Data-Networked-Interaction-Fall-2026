# Week 10 – Homework

## Assignment 10: Ambient Intelligence — Environmental Data as Continuous Aesthetic Material

**Duration:** 1.5 Weeks (Week 10–11) · **Tools:** p5.js · Tone.js · fetch() · Environmental APIs

### Overview

Design a system that runs unattended—no user interaction required—and transforms live environmental data (weather, seismic activity, air quality, tide height, ISS position, aurora index) into a continuous audiovisual environment. You are designing behavior over time, not a single state.

---

### Undergraduate Requirements

- Fetch live environmental data from at least 2 APIs; update every 5 minutes or faster if the API allows.
- Design a visual and sonic system that changes smoothly as data changes (no jarring jumps).
- The piece should run for 30 minutes without repetition; show time-lapse documentation.
- Include a data provenance layer: display source, last update time, and raw values as a subtle overlay.
- Write 250 words: how does designing for 'no interaction' change your design decisions?

### Graduate Extensions

- Integrate at least 4 environmental data streams; design a unified 'world state' object that synthesizes them.
- Implement predictive smoothing: interpolate between API updates using data trends rather than raw jumps.
- Design for gallery installation: the system should recover automatically from network failures and browser crashes.
- Create a 'seasons' feature: the aesthetic shifts based on time of day and time of year.
- 600-word essay: situate your work in the tradition of ambient art (Eno, Turrell, Eliasson) and discuss data as atmosphere.

---

### Deliverables

- Hosted piece
- 5-min screen recording
- Reflection essay (250 words undergrad / 600 words grad)

### Critique Criteria

Temporal design, data–aesthetic coherence, robustness, documentation
