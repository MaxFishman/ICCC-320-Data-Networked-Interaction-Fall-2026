# Week 07 – Homework

## Assignment 07: Data Portrait — Designing a Personal Data Visualization System

**Duration:** 2 Weeks (Week 7–8) · **Tools:** p5.js · fetch() · JSON · Any personal data API

### Overview

What does your digital life look like as form? Collect personal data—steps, sleep, Spotify listening history, GitHub commits, screen time, messages sent—and design a visual system that reveals something true and unexpected about it. This is an assignment in both data literacy and self-representation.

---

### Undergraduate Requirements

- Collect at least 30 days of personal data from one or more sources (manual CSV, API, or export).
- Design a custom visual encoding (not a bar chart or scatter plot—invent a form).
- The visualization must be dynamic: update when new data arrives or when the user interacts.
- Include a written legend explaining what every visual variable encodes.
- Reflect in 300 words: what did the data reveal that surprised you?

### Graduate Extensions

- Integrate data from at least 3 heterogeneous sources; design a unified schema that reconciles their differences.
- Build a live data pipeline: automate collection, normalization, and rendering without manual exports.
- Design for ambiguity: deliberately leave one dimension of the data unresolved and discuss why in your writing.
- Create a 'data zine'—a 4-page printable PDF alongside the interactive piece—that tells the story of the dataset.
- 800-word critical essay: situate your work in relation to self-tracking culture and quantified-self critique.

---

### Deliverables

- Hosted visualization
- Data legend
- Written reflection (300 words undergrad) or data zine + 800-word essay (grad)

### Critique Criteria

Visual invention, data accuracy, personal insight, schema design (grad)
