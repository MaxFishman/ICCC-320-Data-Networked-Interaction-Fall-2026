# Week 11 – Homework

## Assignment 11: The Network Is the Instrument — Latency, Loss, and Glitch as Compositional Material

**Duration:** 2 Weeks (Week 11–12) · **Tools:** Tone.js · WebSockets · OSC · Web MIDI · Node.js

### Overview

Networks are not neutral infrastructure—they are noisy, lossy, latent, and unreliable. Make a live performance piece where the network's imperfections are foregrounded rather than hidden. Latency becomes rhythm. Packet loss becomes texture. Reconnection becomes event.

---

### Undergraduate Requirements

- Build a live networked audio/visual performance tool using Tone.js and WebSockets.
- The piece must be performed live in class for 3–5 minutes with at least one other device involved.
- Network events (disconnection, latency spikes, message drops) must be audible or visible—do not suppress them.
- Document your performance setup: signal flow diagram, instrument map, performance score or notes.
- Write 300 words: what does it feel like to perform with a network as a collaborator?

### Graduate Extensions

- Design a formal performance score that specifies how network events are interpreted (a 'network score').
- Implement intentional network degradation tools: a delay slider, a packet-drop simulator, a jitter knob.
- The performance must involve at least 3 performers on separate devices following the network score.
- Record the performance; annotate the recording with timestamps noting significant network events.
- 800-word essay: engage with the literature of network music (Jacktrip, Telematic music, LOLA) and position your work.

---

### Deliverables

- Live performance
- Signal flow diagram
- Performance score
- Essay (300 words undergrad / 800 words grad)

### Critique Criteria

Performative risk, network integration depth, score/essay quality, live execution
