# Week 02 – Homework

## Assignment 02: Shape of Sound — Audio Synthesis and Real-Time Parameter Mapping with Tone.js

**Duration:** 1 Week (Week 2–3) · **Tools:** Tone.js · p5.js · JavaScript

### Overview

Sound is data. Build a browser-based instrument where at least two synthesis parameters are driven by live data or user interaction mapped through a deliberate, documented mapping logic. You are designing the relationship between data and timbre—not just triggering notes.

---

### Undergraduate Requirements

- Create a Tone.js synth (any type: `Synth`, `AMSynth`, `FMSynth`, etc.) controllable through mouse or keyboard.
- Map at least two interaction dimensions (e.g., X → frequency, Y → reverb wet) with clear visual feedback.
- Add a `Tone.Reverb` and one additional effect (`Delay`, `Chorus`, `Distortion`, etc.).
- Display the current parameter values numerically on screen.
- Write a 200-word mapping rationale: why did you choose these relationships?

### Graduate Extensions

- Fetch live data (weather, seismic, financial) and map it to at least four Tone.js parameters continuously.
- Design a custom polyphonic instrument with voice management (max voices, note-stealing strategy).
- Implement a `Tone.Sequence` or `Tone.Pattern` driven by JSON array data fetched from an API.
- Build an exportable preset system: serialize all parameters to JSON, allow load/save in the UI.
- Write a 500-word essay situating your instrument within the history of data sonification.

---

### Deliverables

- Hosted sketch
- Parameter mapping diagram
- Written rationale (200 words undergrad / 500 words grad)

### Critique Criteria

Sonic intentionality, mapping clarity, data–sound coherence, documentation
